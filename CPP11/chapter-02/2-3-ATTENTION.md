## 2.3 描述的是符合类型，c++有好几种符合类型
1. 这一节介绍的是引用和指针
2. c++中引入了一种新的引用关系，那就是右值引用，其实普通说的都是左值引用（默认）
3. 一条声明语句，其实是由 基本数据类型 + 声明符  共同组成声明语句

### 引用的声明符是在普通的声明符上加上&符号实现声明的定义
1. 比如：int i= 1024， &a = i; 这就是在给i定义引用别名符号（需要注意的是引用必须初始化，而且要是变量，而非字面值）
2. 记住引用无法二次绑定，所以必须初始化
3. 记住引用即别名，因为不会新开辟空间
4. 2.4.1和15.2.3会介绍两种特例
5. 除了以上两种特例，其他所有类型的引用都必须与之绑定的类型对象严格匹配
6. 而且必须和对象进行绑定，不能与字面值和表达式计算结果进行绑定

### 指针的声明符是在普通的声明符上加上*符号实现声明的定义
1. 因为指针就是一个对象，而是指向对象地址的对象，所以可以修改其绑定
2. 不必立马初始化
3. 指针占用的空间和指向对象的类型无关
4. 引用不是一个对象,所以不能定义指向引用的指针
5. 初始化或者给指针赋值的时候需要地址,所以对于普通变量来讲,需要取地址符号(&)去获取对象的地址.
6. 注意不要将引用定义符号和取地址的同一符号不同意义所吓倒
7. 除了2.4.2和15.2.3中特殊的情况,指针对应的类型必须和变量类型严格对应
8. 需要访问指针所指向的对象的时候,通过指针的解引用*符号进行访问,并可以进行赋值操作.
9. 给指针赋初值,使用nullptr NULL 0
10. c++11建议大家使用nullptr
11. 建议大家都初始化指针,在使用指针之前首先检查指针是否为空

###　加※:指针可以逻辑运算比较,会返回bool值,但是想要比较两个指针是否相等,那么需要保证指针有效
1. 指针相同有三种情况，第一为零　第二指向对象地址　第三指向对象地址下一个位置
2. 需要额外注意的是,指向不同对象的下一个位置也会使得指针相等,所以才要保证指针的有效,才能去比较
3. 3.5.3将再仔细介绍指针相关知识
