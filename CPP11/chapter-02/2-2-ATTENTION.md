### 变量-提供了一个具名的、可供程序操作的空间
1. 每个变量都有其数据类型，数据类型决定了所占内存空间的大小和布局方式、该空间能存储值的范围
2. 还有就是变量的类型可以决定变量可以参与运算的类型
3. 在C++中一般 变量和对象是可以互换使用的

### 变量初始化
1. 局部变量尽量在定义时候初始化
2. 类成员尽量在构造函数中进行初始化
3. 而且在c++中初始化和赋值是两个操作，不要搞混了（后续会加强这点的描述）
4. 需要明确的是，初始化是创建变量然后赋予一个初始值，但是赋值就是将旧的值擦出，然后赋予新的值
5. c++11中有一个新的特点就是list初始化，列表初始化。（因为c++定义了好不几种不同的形式，随意才造成了复杂）读者可以选择一种方便的
6. 比如想要初始化一个int变量，以下的语句都可以做到这一点
int units_sold = 0;
int units_sold = {0};
int units_sold{0};
int units_sold(0);

可以说很迷惑了

### 需要注意的是，以上一种list初始化的方式也可以用相同的语法实现赋值运算.所以算得上是很出名的初始化和赋值方式了
### 加星: 值得注意的地方是, 我们在使用列表初始化的时候,对于内置类型来说,当初始值存在精度损失的时候,编译器就会报错.这样就会降低我们写程序的错误
### 所以说这种情况还是挺有用的方法,所以在实现的代码的时候,我们尽量使用列表初始化方式初始化内置类型


###　接下来就是默认初始化了
1. 如果定义在函数外面的内置类型，就会被初始化为具体内置类型值
2. 如果定义在函数里面里面的内置类型，则不会被初始化，则也不能被直接使用，否则就会报错使用了未初始化的局部变量（错误	C4700	使用了未初始化的局部变量“a”）
3. 就以上教训，局部变量一定要初始化。或者使用之前就行赋值操作
4. 一些类会要求构建对象时需要显示初始化，如果不指定就会报错。如果没有指定，那就看类的值是什么了（未知）


### 变量声明和定义的关系
1. C++最重要的就是分离式编译，也就是多个cpp文件可以最后编译生成多个.obj对象，再经过链接最后生成可执行文件
2. 为了支持分离式编译，在变量申请空间的可以有两种方式：声明和定义
3. 声明是使用extern关键字指定，不能显示初始化，这样才是一个声明，如果初始化了，那么声明extern关键字就失效了，变为定义语句了
4. 直接使用类型 加 变量名的方式就是声明加定义
5. 关键的一点就是分离式编译时候，不同编译单元想使用其他编译单元的符号时候，可以使用到这个声明技术：就像我们使用std::cin和std::cout一样
6. 一般在一个编译单元中要使用另一个编译单元中的符号，必须先包含改单元中符号的声明，一般是放在头文件中
7. 最重要的是变量可以被声明很多次，但是只能被定义一次
8. 还有就是在函数内部试图初始化一个由extern标记的变量时候，就会引发错误
9. 在c++中，变量只能被定义一次，多次定义冲突的话会报错的
10. 如果你想在多个源文件中使用同一个变量，那么声明和定义必须独立，而且只有一个文件才能做定义，其他文件只做声明，不能做定义（但是得包含到它的原始声明）
11. 经过程序测试，因为当你将所有编译单元的cpp文件的中间输出结果，也就是obj文件放在同一个文件夹下，那么你可以在其他编译单元中
使用其中一个编译单元的全局变量定义，前提是你必须通过extern关键字声明那个变量，才可以使用。（但是我觉得一般不推荐这样做）
加※：在c++中一般头文件中是声明，源文件中是定义：
1. 而且一般如果要明确函数定义不能为内敛的话，还是将成员函数放在cpp文件中 而不是在h文件的类定义中
2. 模板类要放在h文件中进行定义（这是c++规定）



### 标识符，也就是变量的命名规则
1. 以下划线和数字、字母组成的
2. 以下划线或者字母开头
3. 用户自定义不能以双下划线开头、也不能以下划线开头接着大写字母开头
4. 定义在函数体外的标识符不能以下划线开头
5. 类名一般大写字符卡头命名
6. 全局变量全大写
7. 变量名字意义发生变化的，需要使用下划线或者驼峰命名法

### c++作用域：在全局作用域中的函数，变量、类型，都是可以被访问到的
1. 局部作用域会覆盖全局作用域的同名变量
2. 如果想要指定全局作用域的变量，则需要使用::符号来说明指定
3. 一般不建议有了全局作用域的变量，还在局部定义相同变量的名字
4. 嵌套内部作用域可以随意使用外部作用域的变量，但是除了作用域就不可以使用该变量了
